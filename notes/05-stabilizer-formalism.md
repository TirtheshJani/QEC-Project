# 05 — The stabilizer formalism

The single most useful idea in QEC. Once you absorb it, every code in the rest of this repo (repetition, Shor, Steane, surface code) becomes "pick a stabilizer group and read off the rest". The point of this note is to motivate the formalism, lay out its bookkeeping rules, and connect it to the tableau simulator we just built (`qec/stabilizer.py`).

## The Pauli group on n qubits

The n-qubit Pauli group P_n is the set of all tensor products of single-qubit Paulis with an overall phase in {±1, ±i}. Two facts:

1. **Closed under multiplication.** P · Q ∈ P_n for any two Paulis. The hard part of this is just bookkeeping: `XZ = -iY`, `ZX = iY`, etc. — the multiplication rules of the single-qubit Paulis.

2. **Symplectic representation.** Drop the phase. Each non-trivial single-qubit factor I/X/Y/Z is encoded by two bits `(x, z)`:
   ```
   I = (0, 0),   X = (1, 0),   Y = (1, 1),   Z = (0, 1).
   ```
   So an n-qubit Pauli (modulo phase) is a binary vector in `Z_2^{2n}`. Multiplication of two phaseless Paulis becomes XOR of these bit-vectors. **Commutation** of P and Q reduces to a single bit: they commute iff
   ```
   omega(P, Q) := x_P · z_Q  +  z_P · x_Q  (mod 2)  =  0.
   ```
   This is the **symplectic inner product**. It collapses a question that would otherwise need explicit matrix multiplication into one bit-vector dot product.

`qec/pauli.py` is exactly this: a `Pauli` class with `(x, z, phase)` and the multiplication / commutation rules in a few lines of bit arithmetic. The `matrix()` method exists only as a cross-check for tests.

## Stabilizer states and stabilizer groups

A **stabilizer group** S is an abelian subgroup of P_n that does *not* contain `-I`. (If it did, then for any |psi> with `s |psi> = |psi>` for all `s ∈ S`, we'd also have `-|psi> = -I|psi> = |psi>`, contradiction.)

If `S` has `n - k` independent generators on `n` qubits, then the simultaneous `+1` eigenspace of all generators is **2^k-dimensional**. The vectors in this subspace are called the **stabilizer states** for `S`, and the subspace is the **codespace** of the corresponding code.

In particular:
- `n - k = n` independent generators → 2^0 = 1-dimensional → unique state. This is what the tableau simulator tracks.
- `n - k < n` generators → a 2^k-dim *code*. Phases 4-5 use this for [[3,1,d]], [[7,1,3]], [[9,1,3]].

**Why "stabilizer"?** Each generator `s` divides Hilbert space into `±1` eigenspaces of equal dimension. Asking for `s|psi> = +|psi>` halves the space. n−k commuting generators (each cutting in half) yield `2^n / 2^{n-k} = 2^k` dimensions surviving — and crucially, *no information about the encoded state* is leaked when measuring the generators (they all map the codespace to itself).

This is the core trick: measuring a stabilizer tells you *whether* an error has happened (it flips a generator from `+1` to `-1` precisely when an anticommuting Pauli has occurred) without telling you anything about the logical contents.

## Clifford gates

The **Clifford group** `C_n` is the set of unitaries that map Paulis to Paulis under conjugation: `U ∈ C_n` iff `U P U^dagger ∈ P_n` for every `P ∈ P_n`. Equivalently, `C_n` normalises the Pauli group.

The Clifford group is generated by `{H, S, CNOT}`. Their action on Paulis is the entire content of the formalism:

```
H X H = Z         H Z H = X         H Y H = -Y
S X S^dag = Y     S Z S^dag = Z     S Y S^dag = -X
CNOT (X_c I_t) CNOT = X_c X_t       CNOT (I_c X_t) CNOT = I_c X_t
CNOT (Z_c I_t) CNOT = Z_c I_t       CNOT (I_c Z_t) CNOT = Z_c Z_t
```

Memorise the four CNOT identities. They say: under CNOT, an `X` on the *control* propagates to the target; a `Z` on the *target* propagates to the control. Errors flow against the data flow. This is the single most important fact for understanding fault tolerance.

**Gottesman-Knill theorem.** A circuit consisting only of Clifford gates and Pauli measurements, applied to an initial product state of computational-basis states, can be simulated classically in time polynomial in the number of qubits and gates. Quantum advantage requires non-Clifford gates (e.g. `T`).

This is both wonderful (we can simulate large stabilizer codes efficiently) and a constraint (Cliffords alone are not universal — we need `T`, which we'll only get back via magic state distillation in Phase 7).

## The tableau

How do we represent a stabilizer state on a computer? We track a generating set for `S`, plus a "destabilizer" generating set that lets us read out any logical observable. Aaronson and Gottesman (2004) introduced the now-standard data structure:

- `2n + 1` rows by `2n + 1` columns of bits.
- Rows `0..n-1` are **destabilizer** generators; rows `n..2n-1` are **stabilizer** generators; row `2n` is a scratch row used during measurement.
- The first `n` columns are the `x` bits; the next `n` columns are the `z` bits; the last column is the sign bit `r ∈ {0, 1}` (representing `(-1)^r`). Each row encodes one Pauli.

Clifford gates update the tableau with constant-time rules per row (H, S, CNOT each touch only the columns of the gate's qubits). The full description is in Aaronson-Gottesman §4 and is what `qec/stabilizer.py` implements line-for-line. Key methods:

- `StabilizerState(n)` — initialises to `|0...0>`, stabilised by `Z_q` on each qubit.
- `.h(q) / .s(q) / .cnot(c, t)` — Clifford updates.
- `.measure(q)` — Z-basis projective measurement. The algorithm has two cases: "random" (a stabilizer anticommutes with `Z_q`) and "deterministic" (none does).

The complexity is `O(n^2)` per Clifford gate and `O(n^3)` per measurement — polynomial, regardless of how many qubits.

## Why the simulator is exact for QEC

QEC noise models that consist of Pauli errors followed by Pauli measurements (the standard "circuit-level Pauli noise" model) are entirely within the Gottesman-Knill regime. A stabilizer simulator can:

- Encode a logical state in any stabilizer code (just declare the right initial generators).
- Apply syndrome-extraction circuits (which are Clifford).
- Sample syndromes by stochastically choosing an error from the noise distribution.
- Measure stabilizers and read off syndromes.

All exactly, all in polynomial time. This is what `Stim` (Gidney) does at industrial scale; we'll only ever do it on n ≤ 9 qubits for pedagogical reasons.

The non-Pauli noise (e.g. amplitude damping) breaks this — we have to either Pauli-twirl (replace the channel with its closest Pauli channel by averaging) or fall back to density-matrix simulation. For Phases 4-5 we will Pauli-twirl, because the codes we study correct Pauli errors anyway.

## A worked stabilizer: |Phi+>

The Bell state `(|00> + |11>) / sqrt(2)` is stabilised by `XX` and `ZZ`:
- `XX |Phi+> = (X|0> X|0> + X|1> X|1>) / sqrt(2) = (|11> + |00>) / sqrt(2) = |Phi+>`. ✓
- `ZZ |Phi+> = (|00> + |11>) / sqrt(2) = |Phi+>` (both qubits flip sign together). ✓

So `S = {I, XX, ZZ, -YY}` (the closure under multiplication, with `-YY` because `XX · ZZ = -YY` from `XZ = -iY`). The tableau for `|Phi+>` after `H(q0); CNOT(0, 1)` has stabilizer rows `XX` and `ZZ` and destabilizer rows `IX` and `XI` — see `demos/03_stabilizer_demo.ipynb` for the actual tableau printout.

The formalism is the *mechanism*; the next four phases use it for *codes*:

- **Phase 4 (`notes/06`)** — repetition code. Stabilizers `Z_0 Z_1`, `Z_1 Z_2`. The stabilizer formalism makes this trivial.
